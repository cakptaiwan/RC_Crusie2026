# 皇家旅人 Royal Cruiser — 文章審查標準

> Review Agent 審查依據。每篇文章滿分 25 分，18 分以上合格發布，未達標打回修改（最多 3 輪）。

---

## 五個審查維度（各 5 分）

### 1. 立場明確（5 分）

**標準**：文章有具體立場與結論，不以「各有優缺點，視個人需求而定」作為主要結論。

| 分數 | 說明 |
|------|------|
| 5 | 全文立場鮮明，結論具體可引用 |
| 3 | 有立場但部分段落流於中立模糊 |
| 1 | 結論四平八穩，讀者無法帶走明確觀點 |

**常見失分**：結尾用「依個人喜好選擇」、「兩者各有千秋」收場。

---

### 2. 具體數字條件（5 分）

**標準**：文章含有實際數字，包括價格、百分比、時間、數量等可查證資訊。

| 分數 | 說明 |
|------|------|
| 5 | 每個主要論點都有數字支撐 |
| 3 | 有數字但集中在某一段，其餘段落模糊 |
| 1 | 全文缺乏具體數字，用「較便宜」「節省不少」等模糊描述 |

**合格範例**：「提前 48 小時預約可省 23%」、「單人費用約 NT$890–1,450」
**不合格範例**：「提前預約通常比較划算」、「費用視船型而定」

---

### 3. 情境案例（5 分）

**標準**：文章有具體適用情境或目標族群描述，讀者能判斷「這篇是否適合我」。

| 分數 | 說明 |
|------|------|
| 5 | 明確點出 2 種以上適用情境或族群，並給出對應建議 |
| 3 | 有提到適用族群但描述籠統 |
| 1 | 無情境描述，文章像通用說明書 |

**合格範例**：「帶學齡前小孩的家庭首選 Solarium Bistro，早鳥時段（7:30–9:30）無排隊壓力」
**不合格範例**：「適合各種旅客」、「不同需求的旅客可依喜好選擇」

---

### 4. 明確建議（5 分）

**標準**：文章提供讀者可直接執行的具體行動建議。

| 分數 | 說明 |
|------|------|
| 5 | 至少 3 條可直接執行的建議，含時間點或操作步驟 |
| 3 | 有建議但過於原則性，缺乏操作細節 |
| 1 | 無明確建議，或建議過於抽象（「多做功課」「提早規劃」）|

**合格範例**：「登船第一天中午前往 Chops Grille 現場訂位，可拿到本航次最優惠桌位」
**不合格範例**：「建議旅客提早安排用餐計畫」

---

### 5. 資訊密度（5 分）

**標準**：每個段落都提供新資訊，無重複、填充或廢話段落。

| 分數 | 說明 |
|------|------|
| 5 | 全文段落各司其職，無重複資訊 |
| 3 | 有 1–2 個段落明顯在重複前文或填充字數 |
| 1 | 多處重複，或有明顯湊字數的過渡段落 |

**常見失分**：開頭大段解釋「遊輪是什麼」、結尾重複列一次前文重點。

---

## 合格標準

| 總分 | 判定 |
|------|------|
| 22–25 | 優秀 |
| 18–21 | 合格 |
| 14–17 | 不合格 |
| 13 以下 | 不合格 |


## 附註

- 審查對象：Notion Page Content（Blocks JSON）
- 語言：繁體中文，不接受簡體
- 語氣檢查：不得出現業配語氣（「強烈推薦」「絕對必買」「超值優惠」）
